## 1. 서비스 개요

### TryAngle이란?

**"내가 원하는 사진을 보여주면, AI가 똑같이 찍는 법을 알려준다"**

### 핵심 가치

사진 촬영에서 가장 어려운 것:
- ❌ "어떤 각도로 찍어야 하지?"
- ❌ "얼마나 가까이 가야 하지?"
- ❌ "어떤 포즈를 취해야 하지?"
- ❌ "12번 찍었는데 다 별로야..."

TryAngle은:
- ✅ 레퍼런스 사진을 보여주면
- ✅ AI가 실시간으로 가이드
- ✅ "카메라를 10cm 낮춰주세요"
- ✅ "왼팔을 15도 더 올려주세요"
- ✅ **3번 안에 완벽한 사진 완성**

### 타겟 사용자

- **데이트 중 사진 못 찍어서 스트레스받는 남자친구**
- **여행지에서 인생샷 찍고 싶은 여행자**
- **SNS용 감성 사진 찍고 싶은 20~30대**
- **제품 사진 찍어야 하는 소규모 쇼핑몰**

## 2. 시장 분석 - 기존 카메라 앱들의 문제점

### 현재 인기 카메라 앱 분석

#### 📱 스노우 (SNOW) - 셀카 특화

**주요 기능:**
- 뷰티 필터 (미백, 보정)
- AR 스티커
- 포즈 가이드 (외곽선 표시)

**문제점:**
```
❌ 포즈 가이드가 "외곽선"만 표시
   → 사용자가 알아서 맞춰야 함
   → "얼마나 더 올려야 하는지" 알 수 없음

❌ 실시간 피드백 전혀 없음
   → 찍고 나서야 "아, 틀렸네" 확인

❌ 특정 상황 전용 포즈만
   → 벚꽃, 할로윈 등 이벤트성
   → 일상 촬영에는 사용 불가
```

**실제 사용 경험:**
```
1. 포즈 선택 (벚꽃 배경)
2. 외곽선만 보임
3. "이 정도면 맞는 것 같은데?"
4. 촬영
5. 확인: "어? 팔 각도가 틀렸네..."
6. 다시 촬영 (×10회)
```

#### 📱 소다 (SODA) - 자연스러운 보정

**주요 기능:**
- 자연스러운 뷰티 필터
- 실시간 보정 미리보기
- 다양한 필터

**문제점:**
```
❌ 촬영 가이드 기능 전혀 없음
   → 100% 사용자가 알아서 촬영

❌ 보정에만 집중
   → 구도, 각도, 포즈 도움 전혀 없음

❌ 찍고 나서 보정하는 방식
   → 이미 망친 사진은 못 살림
```

#### 📱 구닥 (Gudak) - 빈티지 감성

**주요 기능:**
- 필름 카메라 감성
- 제한된 촬영 횟수
- 24시간 후 현상

**문제점:**
```
❌ 촬영 가이드 전혀 없음
❌ 오히려 "잘 찍기 어렵게" 만든 컨셉
❌ 감성은 있지만 실용성 제로
```

#### 📱 Ulike - 전신 셀카

**주요 기능:**
- 전신 사진 보정
- 체형 보정
- 뷰티 필터

**문제점:**
```
❌ 보정에만 집중
❌ 촬영 가이드 없음
❌ 결국 "찍고 보정" 방식
```

#### 📱 iPhone 기본 카메라

**주요 기능:**
- 고품질 촬영
- ProRAW, ProRes
- 그리드 표시
- 포트레이트 모드

**문제점:**
```
❌ 그리드만 있고 가이드 없음
❌ 초보자에게는 너무 복잡
   → ISO? 셔터속도? EV?
❌ 전문가용 기능만 있음
```

### 시장의 공통 문제점

| 카테고리 | 기존 앱들 | 문제 |
|---------|-----------|------|
| **촬영 전** | 포즈 가이드 외곽선만 | "얼마나, 어떻게" 모름 |
| **촬영 중** | 실시간 피드백 없음 | 추측만 가능 |
| **촬영 후** | 보정으로 해결 시도 | 망친 원본은 못 살림 |
| **사용성** | 전문 용어 사용 | 초보자는 이해 못함 |
| **범용성** | 특정 상황 전용 | 일상 사용 불가 |

### 시장의 빈틈 (Gap)

```
┌────────────────────────────────────────┐
│  "촬영 순간"을 도와주는 앱이 없다!     │
│                                        │
│  모두가 "촬영 후 보정"에만 집중        │
│  정작 "잘 찍는 법"은 아무도 안 가르쳐줌│
└────────────────────────────────────────┘

← TryAngle이 여기를 공략 ✅
```

## 3. TryAngle만의 차별점

### 핵심 차별점 4가지

#### 1️⃣ 실시간 구체적 피드백 (유일무이)

**기존 앱 (스노우):**
```
[외곽선만 표시]
👤 ← 사용자가 추측

결과:
- "이 정도면 맞나?"
- 10회 재촬영
- 여전히 불만족
```

**TryAngle:**
```
📐 "카메라를 10cm 낮춰주세요"
💪 "왼팔을 15도 더 올려주세요"
📍 "왼쪽으로 30cm 이동하세요"

현재 완성도: 75% → 85% → 95% → 100%! ✨

결과:
- 정확히 무엇을 해야 하는지 앎
- 3회 이내 완성
- 만족도 극대화
```

**차이:**
- 기존: **"대충 맞춰봐"** (추측)
- TryAngle: **"정확히 이렇게 해"** (수치)

#### 2️⃣ 적응형 포즈 비교 (특허 가능 기술)

**기존 시스템의 치명적 문제:**
```
레퍼런스: 전신 포즈 (머리~발끝)
현재 촬영: 상반신만 보임 (머리~허리)

기존 앱:
❌ "다리 포즈가 틀렸습니다"
   (다리가 안 보이는데요?)

사용자: "???"
```

**TryAngle의 AdaptivePoseComparator:**
```
레퍼런스: 전신 포즈
현재 촬영: 상반신만 보임

TryAngle:
✅ 자동으로 "상반신만" 비교
✅ 다리 피드백 제외
✅ 보이는 부분만 정확히 가이드

"어깨를 5도 내리세요"
"고개를 왼쪽으로 10도"

사용자: "오! 이게 맞네!"
```

**핵심 기술:**
- 포즈 타입 자동 감지 (전신/상반신/얼굴)
- 공통 keypoints만 비교
- 불가능한 요구 절대 안 함

#### 3️⃣ 초보자 친화적 언어 변환

**기존 앱 (iPhone 카메라):**
```
EV +0.7
ISO 400
f/2.8
1/250s

초보자: "이게 뭔 말이야? 😵"
```

**TryAngle:**
```
"화면을 위로 슬라이드해서 밝게 하세요"
"설정에서 ISO를 400으로 바꾸세요 (빛이 부족할 때)"
"조리개를 2.8로 하면 배경이 흐려져요"

초보자: "아! 이렇게 하면 되는구나! 😊"
```

**변환 시스템:**
| 전문 용어 | → | 초보자 언어 |
|----------|---|------------|
| EV +0.7 | → | 화면 위로 슬라이드 (밝게) |
| ISO 400 | → | 어두운 곳에서 ISO 높이기 |
| f/2.8 | → | 배경 흐리게 (인물 강조) |
| 더치 틸트 | → | 카메라를 5도 오른쪽으로 기울이기 |

#### 4️⃣ 일상 범용성

**기존 앱 (스노우 포즈 필터):**
```
사용 가능 상황:
- 벚꽃 축제 🌸
- 할로윈 🎃
- 크리스마스 🎄

일상 촬영:
❌ "카페에서 사진?"
❌ "길거리 전신샷?"
❌ "프로필 사진?"
→ 포즈 없음
```

**TryAngle:**
```
사용 가능 상황:
✅ 카페 셀카
✅ 길거리 전신 사진
✅ 여행지 인증샷
✅ 프로필 사진
✅ 제품 촬영
✅ 데이트 기념 사진
✅ 일상의 모든 순간

+ 사용자가 직접 레퍼런스 추가 가능
  → 무한 확장
```

## 4. 핵심 기능

### (1) 레퍼런스 기반 촬영

**작동 방식:**
```
1단계: 원하는 사진 선택
   ↓
2단계: AI가 0.2초 만에 분석
   - 구도 (3분할, 황금비율)
   - 카메라 앵글 (하이/로우)
   - 포즈 (133개 관절)
   - 인물 크기
   ↓
3단계: 실시간 촬영 가이드
   "현재 60% 완성"
   "카메라를 10cm 낮춰주세요"
   ↓
4단계: 100% 달성!
   "완벽합니다! ✨"
```

### (2) 실시간 완성도 표시

**화면 구성:**
```
┌──────────────────────────────┐
│ 카메라 뷰                     │
│                              │
│  [레퍼런스 썸네일]           │
│                              │
│  ████████░░ 75%              │
│                              │
│  📐 "카메라를 10cm 낮춰주세요"│
│                              │
│  ✅ 포즈  ✅ 위치            │
│  ⏳ 앵글  ⏳ 프레이밍         │
└──────────────────────────────┘
```

### (3) 6단계 프레이밍 분석

**자동 구도 타입 분류:**
1. **전신** (fullBody) - 머리부터 발끝까지
2. **무릎샷** (threeQuarter) - 무릎 위까지
3. **허리샷** (halfBody) - 허리 위까지
4. **상반신** (upperBody) - 가슴 위까지
5. **헤드샷** (headAndShoulder) - 어깨까지
6. **클로즈업** (closeUp) - 얼굴만

**구체적 액션 제안:**
- "화면을 1.5배 확대하세요"
- "카메라를 2걸음 뒤로"
- "비율을 4:3으로 변경하세요"

### (4) 우선순위 기반 피드백

**6가지 카테고리 (중요도 순):**
1. 💪 포즈 (1순위) - 가장 눈에 띄는 요소
2. 📍 인물 위치 (2순위)
3. 🔍 프레이밍 (3순위)
4. 📷 카메라 앵글 (4순위)
5. 🎨 구도 (5순위)
6. 👀 시선 (6순위)

**화면에는 가장 중요한 것 1개만:**
```
현재 60% 완성

📐 "카메라를 10cm 낮춰주세요"

← 이것만 보이고
   나머지는 숨김
```

### (5) 카메라 비율 시스템

**3가지 비율 지원:**
- 4:3 - 전체 화면 (기본)
- 1:1 - 정사각형 (인스타그램)
- 16:9 - 와이드 (유튜브)

**비율별 가이드:**
- 캡처 영역 명확히 표시
- iPhone 기본 카메라와 동일한 UX

## 부록: 기술 스택

### AI 모델 (간략)

**온디바이스 AI:**
- YOLO11s-pose: 인물 검출 (18.2MB, CoreML)
- RTMPose: 133개 관절 (218MB, ONNX)
- Vision Framework: 얼굴 분석 (Apple 네이티브)

**알고리즘:**
- AdaptivePoseComparator: 적응형 포즈 비교 (특허 가능)
- CompositionAnalyzer: 구도 분석 (3분할, 황금비율)
- CameraAngleDetector: 앵글 감지 (하이/로우)
- FramingAnalyzer: 6단계 프레이밍
- GapAnalyzer: 차이 계산 및 우선순위

**처리 플로우:**
```
카메라 프레임 (30fps)
  ↓ 150-200ms
YOLO11s (~50ms) → 인물 검출
  ↓
RTMPose (~150ms) → 133 keypoints
  ↓
Rule Engine (~10ms) → 구도/앵글 계산
  ↓
Comparison (~5ms) → 피드백 생성
  ↓
UI 업데이트 (5~7 FPS)
```

**성능:**
- 총 처리 시간: 150-200ms (33% 개선)
- 메모리: 236MB (25% 감소)
- 배터리: 정상 범위
